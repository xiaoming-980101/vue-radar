Codex 趣味用法

这不是“让 AI 写代码”的传统清单，而是把 Codex 当成会读项目、会看浏览器、会整理电脑和会复盘流程的工作伙伴。 每张卡都给你一个能直接复制的安全提示词，适合先从小任务试起。

Use cases 12

电脑整理、网页跑腿、浏览器 QA、游戏原型、项目规则、长任务推进和提交前审查。

01 先只读检查

涉及文件、账号、付款、发送、删除和系统设置时，先让 Codex 列风险。

02 再给最小目标

不要一上来要完整方案，先让它跑一个可验证的小步骤。

03 关键动作要确认

能自动做的事情很多，但越接近真实账户和真实数据，越要留人工确认。

01 · Case Board

12 个可以马上试的 Codex 用法

每个提示词都按“角色、边界、输出物、确认点”写，复制后把你的具体页面、项目或电脑环境补进去即可。

01电脑网络

排查网速和连接问题

适合让 Codex 先检查 DNS、代理、Wi-Fi、测速结果和系统网络状态，再给出从保守到激进的建议。

请你作为网络排查助手，先只读检查当前网络状态、DNS、代理、Wi-Fi、路由器线索和测速结果，列出可能影响网速的原因。把建议按“无风险、低风险、需要确认”分组；任何修改系统网络配置、重置网络或更换代理的动作，都必须先说明影响并等我确认。
相关能力
02硬盘整理

找出可以释放的磁盘空间

适合清理缓存、大文件、下载残留、构建产物和重复文件，但先做清单，不直接删除。

请你作为硬盘空间整理助手，先只读分析磁盘占用，列出缓存、大文件、下载残留、日志、构建产物和重复内容。每一项都写清预计可释放空间、用途、删除风险和恢复方式；不要删除、移动或压缩任何文件，等我确认具体条目后再执行。
审批机制
03插件排障

定位浏览器或桌面控制卡住

适合排查 Browser、Computer Use、扩展、后台进程、锁文件和日志，让问题先被说清楚。

请你作为 Codex 插件排障助手，先检查当前插件状态、浏览器连接、后台进程、日志和可能的锁文件，判断 Browser 或 Computer Use 为什么连不上。不要直接杀进程、清缓存或重启电脑；先按可能性排序，给我一个安全恢复步骤表。
桌面操作
04客服跑腿

帮你等客服和整理回复

适合长时间排队、退款咨询、售后跟进。Codex 可以盯状态、总结上下文、准备草稿。

请你作为客服流程助手，帮我观察这个客服页面，定时检查是否轮到我，并把历史对话整理成简短上下文。你可以帮我拟定回复草稿，但涉及发送消息、同意条款、取消订单、退款、付款或承诺条件时，必须先停下来让我确认。
Computer Use
05网页流程

让 Codex 跑一遍网页流程

适合表单、后台、预约、资料提交、配置面板这类步骤多、容易漏的流程。

请你作为网页流程助手，先只读浏览这个页面，理解每一步的目标、必填项和风险点。可以帮我填写草稿、整理待办和截图说明，但涉及提交、付款、发送、邀请、删除、发布或更改真实数据时，必须先让我确认。
Browser Use
06游戏原型

把一句玩法变成小游戏

适合用 Phaser、Canvas 或普通 Web 技术快速验证核心循环，而不是一开始追求完整游戏。

请你作为小游戏原型导演，把我的一句话玩法拆成核心循环、操作方式、胜负条件、最小素材和验收标准。先实现一个浏览器里能玩的 Web Demo，再自己试玩一轮，记录卡点、截图或控制台问题，并修到可以分享。
Phaser
07真实 QA

自动点击检查网页问题

适合本地页面、响应式、表单、弹窗、导出、空状态和错误状态的真实浏览器验收。

请你作为本地网页 QA 助手，打开我的本地页面，围绕这次改动完成真实点击测试。请覆盖桌面和手机宽度，记录 bug、截图位置、复现步骤、影响范围和建议修复顺序；先报告问题，再动手修最高优先级的项。
Playwright Skill
08项目记忆

生成 AGENTS.md 项目说明

适合把运行方式、代码边界、验证命令和不该擅自做的动作写给下一次 Codex。

请你作为项目规则整理助手，阅读 README、package.json、配置文件、目录结构和常用脚本，生成一份适合 Codex 读取的 AGENTS.md。内容包括项目目标、运行方式、测试/构建命令、代码风格、风险边界、提交前检查和不能擅自执行的动作。
AGENTS.md
09Skill 工作流

给 Codex 选择合适技能包

适合把常见流程沉淀成可触发的 Skill，减少每次重复解释背景和验收方式。

请你作为 Skill 工作流顾问，判断这个任务适合启用哪些 Skill。优先考虑需求澄清、计划、浏览器验收、TDD、代码审查、文档整理和发布检查；每个 Skill 都要说明它避免什么返工，以及什么时候不应该启用。
Agent Skills
10长任务

让 Codex 分阶段推进大任务

适合需求清楚但步骤很多的任务，让 Codex 持续执行，只在关键风险点回来问你。

请你进入长任务推进模式，以这个目标为准持续推进。先写完成标准、阶段计划和最小可验证步骤，再开始执行；遇到费用、权限、破坏性操作、真实账户、删除数据或无法判断的产品决策时，暂停并向我确认。
Superpowers
11提交前审查

让 Codex 先做代码审查

适合提交前找风险、回归、缺测试和用户影响，先审查，再决定要不要修。

请你作为代码审查助手，先阅读这次改动的 diff，再按 bug 风险、用户影响、缺失测试、可维护性和安全边界排序列出问题。不要直接重写代码，先给审查结论；如果没有明显问题，也要说明剩余风险和未覆盖的验证。
Code Review Skill
12调试底座

搭一套稳定网页调试流程

适合本地应用经常要查控制台、截图、移动端、网络和失败恢复的项目。

请你作为浏览器调试底座助手，为这个项目设计一套稳定的网页调试流程：包含本地启动方式、浏览器连接方式、控制台日志读取、桌面/移动端 QA 清单、截图保存位置和失败恢复步骤。最后用一次真实页面验证流程可用。
Browser Use

02 · Guardrails

最重要的不是让它多做，而是让它会停

Codex 可以替你读页面、跑脚本、整理文件、写代码和做 QA，但这些边界最好写进提示词。

Confirm

真实账户和真实钱

提交、付款、退款、取消订单、邀请成员、发送消息，都要先确认。

Protect

删除和移动文件

先列清单和风险，再让用户确认具体条目，不要整目录一把清。

Scope

长任务先定义完成标准

先写验收条件和最小可验证步骤，避免越做越大。
